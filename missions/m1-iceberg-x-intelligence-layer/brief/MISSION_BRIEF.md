Mission Description

Iceberg X is the internal platform we use to manage R&D missions, internship tasks, mentor assignments, ideas, progress, and project outcomes. This mission focuses on researching how the platform can be improved, identifying missing features or UX problems, and building a working demo for one or more high-value improvements.



The goal is not only to suggest ideas, but to analyse the current R&D workflow, understand what mentors, interns, and admins need, and create a practical proof of concept that could later become part of the real Iceberg X product.



Context & Background

Iceberg X is becoming an important operational system for managing R&D work inside Iceberg Digital. It helps us organise missions, collect ideas, assign mentors, track student/intern progress, and create a more structured innovation pipeline.



As the number of missions, mentors, interns, and project categories grows, the platform needs to become easier to use, more informative, and more useful for day-to-day R&D management.



Potential improvement areas may include:



Better mission discovery and filtering.

Improved mentor assignment workflows.

Clearer mission progress tracking.

Better intern/student dashboards.

Mission difficulty and status visualisation.

Submission and deliverable tracking.

Notifications and reminders.

Better documentation structure.

AI-assisted mission creation.

AI-assisted project evaluation.

Badge, reward, or achievement systems.

Analytics for admins and mentors.

Better Slack integration.

Better repository/demo link tracking.

Improved UX for reviewing completed work.

This mission is valuable because even small improvements to Iceberg X can directly improve how we run R&D, manage interns, evaluate technical work, and turn ideas into working demos.



Problem Statement

We need to investigate how Iceberg X can be improved as an R&D management platform.



The core questions are:



What are the most important pain points in the current Iceberg X workflow?

What do admins need to manage missions more efficiently?

What do mentors need to review and guide projects more easily?

What do interns or contributors need to understand missions and submit work clearly?

Which features would make mission tracking more transparent?

Which parts of the platform can be improved with better UI/UX?

Which workflows can be automated or simplified?

Can AI help with mission creation, evaluation, summarisation, or recommendations?

What kind of dashboard or reporting would help the R&D team?

Which improvement can be demonstrated with a realistic working POC?

The mission should result in both a research document and a demo implementation that proves one or more valuable platform improvements.

Selected POC Feature

Choose one or more high-value improvements and build a working proof of concept.



Suggested POC options:



Option A: AI Mission Generator



Admin enters a rough idea.

System generates mission title, description, context, problem statement, expected deliverables, difficulty level, and category.

Admin can edit and save the result.

Option B: Mission Progress Dashboard



Shows missions grouped by status.

Displays assigned mentors, difficulty levels, categories, and progress.

Highlights blocked, inactive, or high-priority missions.

Option C: Intern Submission Tracker



Allows interns to submit repository links, demo links, notes, and final deliverables.

Allows mentors to review submissions and leave feedback.

Option D: Badge & Achievement System



Defines badges that can be awarded to interns.

Shows earned badges on user profiles.

Supports simple reward and recognition workflows.

Option E: AI Project Review Assistant



User submits project description, repo link, or demo notes.

System generates review questions, strengths, weaknesses, and suggested mentor feedback.
