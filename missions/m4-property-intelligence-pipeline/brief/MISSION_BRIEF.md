Mission Description

plaud.ai is a service that records conversations and generates summaries based on predefined templates. Our customers use these devices during property valuation appointments. We would like to retrieve these summaries from Plaud, determine which property they relate to, and use the information within the property proposal process.



Context & Background

When our customers attend a property valuation appointment, they have detailed conversations with homeowners about a wide range of topics. Instead of manually processing these conversations and entering the information into the system, we want to automatically make use of the captured information within Lifesycle. This will reduce manual work while ensuring that no important details are missed.



Problem Statement

Lifesycle has Companies and Users. When retrieving data from Plaud, we need a reliable way to identify which Company, User, and Property each recording belongs to.



We need to understand whether:



Each Company or User should connect their own Plaud account to Lifesycle, or We should use a central Plaud account and retrieve all recordings from there.



We also need to understand what metadata is available from Plaud and how recordings can be consistently matched to the correct entities within Lifesycle.



Expected Deliverables

A clear understanding of how transcripts can be associated with Companies, Users, and Properties.

Clear documentation explaining how transcripts can be retrieved from Plaud.

A working example or proof of concept, if possible.
